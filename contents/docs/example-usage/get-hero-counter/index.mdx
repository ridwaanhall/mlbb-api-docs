---
title: GET Hero Counter
description: The API endpoint requires a path parameter, main_heroid (an integer from 1 to 127), to fetch counter information for a specific hero. The successful JSON response includes a success code and message and a data field containing a records array. This array holds an object that includes the main hero's name, win rate, appearance rate, and ban rate, along with two lists: sub_hero and sub_hero_last. The sub_hero list contains heroes that perform well against the main hero (counters), each with their win rates, and increase_win_rate, and how their win rate fluctuates throughout the match. The sub_hero_last list contains heroes that are weak against the main hero, each with their respective win rate and how their win rate fluctuates throughout the match. In short, this endpoint provides data on heroes that are effective counters and heroes that are ineffective against the specified main hero, as well as metrics for the selected main hero.
---

## Endpoint

```txt
GET /api/hero-counter/<int:main_heroid>/
```

## Path Parameters

1. **main_heroid**
   The ID of the main hero whose counter information is to be fetched.

   - **Possible Values:** From `1` to `127`
   - **Type:** integer
   - **Required:** `True`

## Example Requests

```txt
GET /api/hero-counter/127/
```

## Example Usage

```python {13} showLineNumbers
import requests
import json

def fetch_api(api_url):
    try:
        response = requests.get(api_url)
        response.raise_for_status()
        return response.json()
    except requests.exceptions.RequestException as e:
        print(f"Error: {e}")
        return None

api_url = "https://api-mobilelegends.vercel.app/api/hero-counter/127/"
data = fetch_api(api_url)

print(json.dumps(data, indent=4) if data else "No data fetched.")
```
