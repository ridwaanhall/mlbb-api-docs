---
title: GET Hero Compatibility
description: The API endpoint requires a path parameter, main_heroid (an integer from 1 to 127), to fetch compatibility information for a specific hero. The successful JSON response includes a success code and message, and a data field containing a records array. This array holds an object that includes the main hero's name, win rate, appearance rate, and ban rate, along with two lists, sub_hero and sub_hero_last. The sub_hero list contains heroes that synergize well with the main hero, along with their win rates and how those rates change over different time segments, and includes each heroes appearance rate, and increase_win_rate The sub_hero_last list contains heroes that do not synergize well with the main hero, also with their win rates and fluctuations throughout the match. In essence, this endpoint provides data on heroes that are good teammates and those that are poor teammates with the specified main hero, along with metrics for the selected main hero.
---

## Endpoint

```txt
GET /api/hero-compatibility/<int:main_heroid>/
```

## Path Parameters

1. **main_heroid**
   The ID of the main hero whose compatibility information is to be fetched.

   - **Possible Values:** From `1` to `127`
   - **Type:** integer
   - **Required:** `True`

## Example Requests

```txt
GET /api/hero-compatibility/127/
```

## Example Usage

```python {13} showLineNumbers
import requests
import json

def fetch_api(api_url):
    try:
        response = requests.get(api_url)
        response.raise_for_status()
        return response.json()
    except requests.exceptions.RequestException as e:
        print(f"Error: {e}")
        return None

api_url = "https://api-mobilelegends.vercel.app/api/hero-compatibility/127/"
data = fetch_api(api_url)

print(json.dumps(data, indent=4) if data else "No data fetched.")
```
